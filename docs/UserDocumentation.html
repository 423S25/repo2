Group 2 User Documentation

Product Info

This project is an inventory management system for the HRDC Warming Hut in Bozeman, Montana. Our goal for this project is to: Create a database to manage the quantities of supplies the Warming Hut uses on a daily basis Alert Warming Hut employees when items reach a certain percentage of inventory.

How to obtain and install the software?

To obtain and install the software, you just need to install docker desktop which already includes the docker and the docker desktop in it. This is for local development and is the only piece of software necessary and then the command "make build" for linux and "docker-compose --env-file .env -f compose.yaml build" for windows. As well ensure that an .env file exists within the base directory for the project. Our website is hosted and is deployed through a CI/CD pipeline that will automatically deploy our software to the NorthFlank web server for each release. For a new developer to be able to do releases you will need access to the NorthFlank server and access to the repo. The application can be deployed by just commiting a change to the repository on the main branch.

How to run?

Before you run the software, you have to make sure that you have docker installed on your computer.

  • Linux: Once you have that, pull all the changes code from the github repository and then run the “make build” command in the terminal. Finally, you can run “make dev” after. As well if a .env file does not exist use "touch .env"
  • Windows: With windows Makefile tools may not be available so run instead "docker-compose --env-file .env -f compose.yaml build" to build and "docker-compose --env-file .env -f compose.yaml up". As well if a .env file does not exist use "New-Item .env".
  • Note: Older versions of docker use "docker compose" instead of "docker-compose" so if docker-compose is not a valid command use "docker compose" instead either in the makefile or as a full command.

How to use the software?

The website consists of a homepage that contains a table of all inventory items. Admin users will also have the option to add new items to the table as well as delete them. Each item in the table will contain the items name, its minimum count, current count, and the price. There will be an analytics page that contains metrics for each item and their use in each month. A user systems page where they can update their info. Finally a notification center that alerts users when certain stock are getting low. These can all be navigated to from the sidebar. The current website can be visited at this url https://p02--hdrc-react-app--sylztdhdybh8.code.run.

How to report a bug?

Bugs will be reported using github issues and the bug templates found here https://github.com/stevemao/github-issue-templates. The issue will include expected behavior, current behavior, possible solution, steps to reproduce, context (Environment), detailed description and possible implementation. There is an issues tab in the repository where any kind of bugs can be reported there. New issues will be created there, where the template can be used and bug reporters can add a title, description and other specific descriptions like labels of the different bugs.

Current Features

  • Inventory table that list the items name, count, minimum count, and category.
  • Users can sort the table by column and search the table
  • User can edit the table by adding, deleting, and editing items in the table
  • Users can check a mock graph of items counts over time.
