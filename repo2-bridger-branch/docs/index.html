Product Info

This project is an inventory management system for the HRDC Warming Hut in Bozeman, Montana. Our goal for this project is to:Create a database to manage the quantities of supplies the Warming Hut uses on a daily basisAlert Warming Hut employees when items reach a certain percentage of inventory.

User Documentation

How to obtain and install the software?

To obtain and install the software, you just need to install docker desktop which already includes the docker and the docker desktop in it. This is for local development and is the only piece of software necessary. We will also use Heroku to host our website and develop a CI/CD pipeline that will automatically deploy our software to the Heroku web server for each release. For a new developer to be able to do releases you will need access to the Heroku server and an installation of git. The application can be deployed using these instructions https://devcenter.heroku.com/articles/git/.

How to run?

Before you run the software, you have to make sure that you have docker installed on your computer. Once you have that, you can just pull all the changes code from the github repository and then run the “make build” command in the terminal. Finally, you can run “make dev” after.

How to report a bug?

Bugs will be reported using github issues and the bug templates found here https://github.com/stevemao/github-issue-templates. The issue will include expected behavior, current behavior, possible solution, steps to reproduce, context (Environment), detailed description and possible implementation. There is an issues tab in the repository where any kind of bugs can be reported there. New issues will be created there, where the template can be used and bug reporters can add a title, description and other specific descriptions like labels of the different bugs.
